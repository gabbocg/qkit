# Spec A — Rename `qbeamer` to `qkit` and Add CV Format

**Date:** 2026-05-28
**Status:** Draft (pre-implementation)
**Successor work:** Spec B will add `qkit-manuscript`. Out of scope here.

## 1. Motivation

The R package currently named `qbeamer` ships a single Quarto extension that produces a Beamer presentation. The owner wants to broaden the package into a small toolkit of useful Quarto building blocks — starting with the existing presentation and a new CV format ported from a private RMarkdown template. The new identity is `qkit` ("a small set of useful Quarto things").

The user's CV currently lives at `~/Dropbox/Documentos/Research/CV/` as `rmarkdown-cv.Rmd` + a full Pandoc-template `latex-cv.tex`. That source is **not** migrated into the package; only a generic, feature-demonstrating skeleton is bundled.

## 2. Goals

1. Rename the R package `qbeamer` → `qkit` with a hard cut (no compatibility shims).
2. Generalize the install/scaffold API so the user picks a format at install or scaffold time.
3. Migrate the existing Beamer extension to `qkit-beamer` with no visual changes.
4. Add a `qkit-cv` Quarto extension that produces a LuaLaTeX CV with the styling features from the user's existing RMarkdown CV.

## 3. Non-goals

- `qkit-manuscript` (Spec B).
- Pkgdown site, GitHub Actions CI, `NEWS.md`.
- Deprecation shims, re-exports, or aliases keeping `qbeamer` callable.
- Visual redesign of the Beamer format.
- Renaming the GitHub repository (the owner handles that on github.com; the spec only updates URLs in `DESCRIPTION`).
- Migrating the user's actual CV content; that stays in the private `Research/CV/` directory.

## 4. User stories

- *As a presenter*, I run `qkit::install_extension(type = "beamer")` in a Quarto project and get the Beamer extension under `_extensions/qkit-beamer/`, just like the old `qbeamer` flow.
- *As a CV author*, I run `qkit::create_project("my-cv", type = "cv")` and get a directory containing `cv.qmd` (a feature-demonstrating skeleton) plus `_extensions/qkit-cv/`. I render with `qkit::qkit_render("my-cv/cv.qmd")` and produce a PDF matching the visual style of the reference CV.
- *As an RStudio user*, `File > New Project` shows two `qkit` entries — one for a Beamer presentation, one for a CV — each with its own wizard.
- *As a kit author*, I can add `"manuscript"` as a third format value in Spec B without changing function signatures.

### 5.2 R API surface

All four exported functions live under the `qkit_*` / verb-noun naming used today:

```r
install_extension(path = ".", type = "beamer", overwrite = FALSE)
```
- `type` is a character vector of `c("beamer", "cv")`. Passing both installs both extensions in one call.
- Each `type` resolves to `system.file("extdata", "_extensions", paste0("qkit-", type), package = "qkit")` and is copied to `<path>/_extensions/qkit-<type>/`.
- If the target directory exists and `overwrite = FALSE`, emit a message and skip.
- Returns invisibly the vector of installed target paths.

```r
create_project(path, type = "beamer", title = "Untitled", ...)
```
- Creates `path`, copies the matching skeleton (`beamer.qmd` or `cv.qmd`) from `inst/rstudio/templates/project/skeleton/` into `path/<type>.qmd` (or a fixed name like `index.qmd` — see §5.5).
- For `type = "beamer"`, substitutes `title` placeholder. For `type = "cv"`, substitutes an `author` placeholder instead (CV has no title).
- Calls `install_extension(path = path, type = type)` to drop the extension files.
- Returns invisibly the project path.

```r
qkit_render(input, ...)
qkit_preview(input, ...)
```
- Thin wrappers around `quarto::quarto_render` / `quarto::quarto_preview`.
- Both call `ensure_extension(input)` first.

```r
ensure_extension(input)   # internal
```
- Reads the YAML front matter of `input` using `yaml::read_yaml()` (or `rmarkdown::yaml_front_matter()` — see §6.2).
- Inspects the `format:` key. If it resolves to `qkit-beamer` or `qkit-cv` and the corresponding `_extensions/qkit-<type>/` dir is missing next to `input`, calls `install_extension(path = dirname(input), type = <type>)`.
- If the format isn't a recognized `qkit-*` value, the function is a silent no-op — Quarto handles the error path normally.

### 5.3 `qkit-beamer` extension

Verbatim relocation of the existing `qbeamer` extension. No content changes other than:
- Directory renamed to `_extensions/qkit-beamer/`.
- `_extension.yml` `title:` field updated from `qbeamer` to `qkit-beamer`.

All other files (`preamble.tex`, `title.tex`) are byte-identical to today.

### 5.4 `qkit-cv` extension

#### 5.4.1 `_extension.yml`

```yaml
title: qkit-cv
author: Gabriel Cabrera Guzmán
version: 0.1.0
contributes:
  formats:
    pdf:
      include-in-header:
        - preamble.tex
      template-partials:
        - before-body.tex
      pdf-engine: lualatex
      geometry:
        - margin=0.70in
      fontfamily: mathpazo
      fontfamilyoptions: [sc, osf]
      fontsize: 10pt
      linkcolor: blue
      urlcolor: blue
      keep-tex: true
```

The format key is `pdf` (not `cv`); users invoke it via `format: qkit-cv` in their YAML and Quarto resolves the extension. This matches the Quarto convention for custom PDF formats.

#### 5.4.2 `preamble.tex`

Ports the "stuff I added" block from the user's existing `latex-cv.tex`, stripped of all Pandoc-template conditionals (Quarto's default template already handles fontspec, hyperref, geometry, etc.). Contents include:

- `\usepackage{multicol}` — two-column References block.
- `\usepackage{marvosym}` — `\Letter`, `\Telefon` glyphs.
- `\usepackage{fontawesome}` — `\faEnvelopeO`, `\faGithub`, `\faGlobe`, `\faPhone` icons in the header.
- `\usepackage{enumitem}`, `\usepackage{calc}` — list customization for hanging-indent publications.
- `\usepackage{array, xcolor}` — custom column types `L` (right-aligned, 0.14\textwidth) and `R` (left-aligned, 0.8\textwidth), plus `\VRule` divider.
- `\usepackage{sectsty}` — section font overrides.
- `\usepackage{titlesec}` — section spacing tuned to `\titlespacing\section{0pt}{12pt}{4pt}`.
- `\usepackage{fancyhdr,lastpage}` — page footer with author/title on the left and `page/total` on the right.
- `\usepackage{parskip}` — paragraph spacing in lists.
- Itemize override (no bullets, indented).

The `pageof` footer references `$author$` and `$title$` from YAML, which Quarto exposes as `\author{}` / `\title{}` macros once a small `\fancyfoot` snippet is included. Pages: `\thepage/\pageref{LastPage}` on the right.

#### 5.4.3 `before-body.tex`

A Pandoc partial that renders the centered name + contact block before the document body. Reads the following YAML metadata:

| Field | Type | Default | Render |
|---|---|---|---|
| `author` | string | required | `\centerline{\huge \bf <author>}` |
| `jobtitle` | string | optional | `\moveleft.5\hoffset\centerline{<jobtitle>}` |
| `address` | string | optional | `\moveleft.5\hoffset\centerline{<address>}` |
| `email` | string | optional | `\faEnvelopeO` + mailto link |
| `phone` | string | optional | `\faPhone` + plain text |
| `github` | string | optional | `\faGithub` + `https://github.com/<github>` link |
| `web` | string | optional | `\faGlobe` + `https://<web>` link |
| `orcid` | string | optional | `\orcidlink{<id>}` + ID (uses `\usepackage{orcidlink}` — add to `preamble.tex`) |
| `fontawesome` | bool | `true` | If `false`, fall back to `E-mail:` / `Phone:` / `Github:` / `Web:` labels |
| `updated` | bool | `false` | If `true`, append `\| Updated: \today` to the contact line |

Each field is conditional (`$if(field)$ … $endif$`), so omitting any one skips its output cleanly.

### 5.7 Dependencies

- `Imports`: `fs` (already present), `yaml` (new — needed by `ensure_extension()` to detect format from YAML front matter).
- `Suggests`: `quarto` (already present).

`yaml` is a base-tier R package with no compiled code and trivial install cost.

## 6. Open questions resolved during brainstorm

- **Spec scope**: One spec covers rename + multi-format + Beamer migration + CV addition. Manuscript is Spec B.
- **Hard vs soft rename**: Hard cut.
- **Format packaging**: Two separate extensions (`qkit-beamer`, `qkit-cv`), not one extension with two formats.
- **CV skeleton flavor**: Feature demo (all LaTeX patterns shown with placeholder content).
- **API shape**: Single `install_extension(type = ...)` / `create_project(type = ...)`, no per-format functions.
- **CV engine**: LuaLaTeX (matches existing user setup; Beamer stays on XeLaTeX).
- **Title strategy**: `before-body.tex` partial reading Quarto YAML, *not* a full Pandoc document template.

## 7. Validation gates

The work is complete when all four pass:

1. `devtools::document()` regenerates `NAMESPACE` and `man/` with no errors.
2. `devtools::check()` returns 0 errors and 0 warnings. Notes are acceptable.
3. `inst/examples/beamer-example.qmd` renders to PDF successfully via `qkit_render()`.
4. `inst/examples/cv-example.qmd` renders to PDF successfully via `qkit_render()` and visually matches the reference CV in: section header style, horizontal rules under each section, `L|R` tabular layout for date/description rows, hanging-indent numbered publications, two-column references block, and contact-icon header.

## 8. Risks and mitigations

| Risk | Mitigation |
|---|---|
| Pandoc default LaTeX template conflicts with custom `preamble.tex` macros | Use `include-in-header` (Quarto loads these *after* Pandoc's preamble), avoid redefining packages Pandoc already loads |
| `fontawesome` LaTeX package requires TeXLive ≥ 2015 | Document this requirement in the format's README/example header; not a code change |
| `mathpazo` font with `sc, osf` options can fail under certain TeXLive distros | Mirrors user's existing working setup; if it breaks for downstream users, they override via YAML |
| User's existing `qbeamer` projects break post-rename | Acceptable per user decision (hard cut, pre-1.0 package) |
| `ensure_extension()` YAML parse fails on malformed front matter | Wrap in `tryCatch`; on parse error, skip auto-install and let Quarto report the error |
